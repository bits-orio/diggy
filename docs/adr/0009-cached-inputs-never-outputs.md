# Stress performance: cache the world's facts, verify anything derived

ADR 0008 made stress a pure function of geometry, and the naive implementation read that geometry through the engine API every time: ~55 `get_tile` calls plus an entity query per cell, measured at ~150–190 µs/cell — ~8 ms per dig, ~30 ms per wall change, a quarter-second freeze for a 30-rock grenade burst. Unacceptable for one busy server, let alone 20 MTS teams.

The ledger we removed was fast for a reason: it was a cache. Its disease was *what* it cached — arithmetic accumulations, where every missed or asymmetric event compounds forever. The soundness rule this ADR adds: **caches may hold facts (assignments) or verified derivations, never bare accumulations.**

Layer 1 — the world mirror (`scripts/lib/mirror.lua`). A chunk-local copy of exactly the facts stress reads: a per-tile code (void / water / floor / flooring tier, plus a rock-cover flag) and per-chunk wall/reactor records. `compute_cell` runs entirely on Lua array reads — zero API calls. Soundness: every write is an assignment flowing through the same choke points that already announce world changes (the dig pipeline, frontier advance, carving, tile events, support build/removal, collapse fill); a missed event leaves one stale code that the next write or chunk rebuild fixes — nothing compounds. The mirror lives in module locals, never in `storage`: chunks rebuild lazily from the engine after load, idle chunks are evicted (two-strike sweep), and 20 idle surfaces cost nothing in RAM or save size. Removal events force-sync their chunk before correcting it, because the engine may still report a dying entity to a mid-event sync. `/diggy-mirror-check` (and the `debug_mirror_check` remote) diffs cached chunks against a fresh engine read — the exactness proof used by the benchmark harness.

Lockstep determinism: a local cache is safe only if every peer holds identical content. The mirror is wiped on every `on_player_joined_game`, so the joiner (who starts empty) and the veterans (reset at the same tick) re-derive identical chunks from identical events.

Layer 2 — the verified stress cache (in `collapse.lua`). Cell stress values are kept incrementally: each mirror change applies a mechanically-derived delta (the ± of the same mask/strength terms `compute_cell` uses, so symmetry is by construction, not by discipline). This is an accumulation — the ledger's shape — so it is never trusted at the moment it matters: any threshold *transition* (warning marker on/off, collapse trigger) is confirmed by a fresh `compute_cell` against the mirror before acting, and a seed-keyed audit recomputes one cell per evaluation, correcting and logging any drift. The cache is wiped on player join (determinism), on Support Struts research (reach changes every wall's contribution), and on configuration change.

Measured (headless, seed 12345, same scenario as the ADR 0008 benchmarks): a full compute drops from ~152 µs/cell (engine reads) to ~17–20 µs/cell (mirror + flat LUTs) — and the cache makes full computes rare. The complete dig pipeline (stress + spawner + veins + charting + frontier) measures ~1.0 ms/dig; an 86-rock single-tick storm totals ~83 ms where the naive engine-read version cost ~8 ms per dig on its own. Wall placement falls from ~30 ms to well under a millisecond (reach-aware radius + delta application). After two full benchmark sims plus the storm: mirror 0 mismatches, cache max delta 0.0000 across 1368 cells, zero audit corrections. Identical geometry still always reads identically — drift can only be a transient performance artifact, never a wrong collapse.
